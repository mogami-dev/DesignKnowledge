# UI Review Checklist

## Layout

- [ ] 余白は十分か
- [ ] 要素は揃っているか
- [ ] 情報量は適切か
- [ ] スクロール量は多すぎないか
- [ ] 視線誘導は自然か

## Color

- [ ] 色数は少ないか
- [ ] Primaryが明確か
- [ ] コントラストは十分か
- [ ] ダークモード対応可能か
- [ ] 危険操作の色が明確か

## Typography

- [ ] フォントサイズは統一されているか
- [ ] 行間は適切か
- [ ] 見出しと本文の強弱があるか
- [ ] 長文が読みやすいか

## Components

- [ ] ボタンサイズは適切か
- [ ] タップ領域は44pt以上か
- [ ] アイコンの意味は分かるか
- [ ] 入力欄は使いやすいか
- [ ] 同じ部品が同じ見た目になっているか

## UX

- [ ] 初見で理解できるか
- [ ] CTAが分かるか
- [ ] 戻る導線があるか
- [ ] エラー表示があるか
- [ ] Loadingがあるか
- [ ] Emptyがあるか
- [ ] 誤操作を防げるか

## Accessibility

- [ ] Dynamic Typeに対応できるか
- [ ] VoiceOverで意味が伝わるか
- [ ] Color Contrastは十分か
- [ ] タップ領域は44pt以上か
- [ ] 色だけに依存していないか

## Platform

- [ ] iOSならApple HIGに沿っているか
- [ ] AndroidならMaterial Designに沿っているか
- [ ] Webならレスポンシブ対応可能か

## Animation

- [ ] 必要最低限か
- [ ] 速すぎないか
- [ ] 違和感がないか
- [ ] 操作理解を助けているか

## Score

100点満点で評価する。

- 90以上: 実装可
- 80〜89: 軽微な修正後に実装可
- 70〜79: 要改善
- 69以下: 再設計推奨
