# Design Review Guide

## Purpose

人間とChatGPTがデザインレビューを行う際の標準手順を定義する。

レビューは主観だけでなく、理由・影響・代替案をセットで示す。

## 入力

- 画面画像または画面説明
- 画面仕様
- 対象ユーザー
- 利用目的
- プラットフォーム
- 実装技術

## 必須レビュー項目

### 1. Layout

- 余白
- 整列
- 視線誘導
- 情報量
- スクロール

### 2. Visual

- 配色
- フォント
- アイコン
- 画像
- 一貫性

### 3. UX

- 理解しやすいか
- 迷わないか
- 主要CTAが明確か
- 誤操作が起きにくいか
- 学習コストは低いか

### 4. Accessibility

- Contrast
- Dynamic Type
- VoiceOver
- 44pt Rule
- 色だけに依存していないか

### 5. Platform

- Apple HIG
- Material Design
- Web Standard

### 6. Performance

- 重そうなUIではないか
- 画像が多すぎないか
- アニメーションが過剰ではないか

### 7. Implementation

- SwiftUIで自然に実装できるか
- Reactで自然に実装できるか
- 状態管理が複雑すぎないか
- 実装コストに見合う価値があるか

## 出力形式

```md
# Design Review

## 良い点

- ...

## 改善点

- ...

## 優先順位

### High

- ...

### Medium

- ...

### Low

- ...

## 修正案

- ...

## 総合評価

Score: xx / 100

## 実装可否

- 実装可 / 修正後実装可 / 再設計推奨
```

## レビュー原則

- 理由を必ず説明する
- 代替案を提示する
- Apple標準やMaterial標準を尊重する
- ユーザー価値を優先する
- 見た目だけでは評価しない
- 実装コストも考慮する
