# Design Playbook

## 画面案作成

```text
この画面を5パターン考えてください。

条件:
- iOS標準
- 余白重視
- シンプル
- 初心者でも迷わない
- 実装しやすい

各案について、特徴・メリット・デメリット・向いているケースを出してください。
```

## デザインレビュー

```text
以下の画面をレビューしてください。

観点:
- UI
- UX
- 情報量
- アクセシビリティ
- プラットフォーム標準
- 実装コスト

100点満点で評価し、High / Medium / Lowで改善案を出してください。
```

## Inspiration比較

```text
この画面について、以下の方向性で比較してください。

- Apple Photos風
- Instagram風
- Notion風
- Spotify風
- Airbnb風

それぞれのメリット・デメリットと、このアプリに合うかを評価してください。
```

## Color提案

```text
サービスコンセプトから配色を提案してください。

出力項目:
- Primary
- Secondary
- Accent
- Background
- Surface
- Text Primary
- Text Secondary

各色について、採用理由と避けるべき使い方も説明してください。
```

## コンポーネント洗い出し

```text
この画面に必要なUIコンポーネントを一覧化してください。

対象:
- Button
- Card
- BottomSheet
- Dialog
- Tab
- Toast
- FAB
- Empty
- Loading
- Error

各コンポーネントについて、状態・表示条件・操作を整理してください。
```

## Codex向け資料化

```text
このデザイン方針を、Codexが実装できる仕様書に変換してください。

含めるもの:
- 画面目的
- レイアウト
- コンポーネント
- 状態別表示
- 操作
- アニメーション
- デザインシステム
- 実装時の注意点
```
