# DesignKnowledge

人間とChatGPTで要件定義・画面設計・デザイン設計を行い、最終的にCodexへ渡すためのデザインナレッジベースです。

実装コードを置くリポジトリではありません。

## 位置付け

- 人間: プロダクト判断・最終意思決定
- ChatGPT: 要件整理、UX/UI案作成、レビュー、設計書化
- Codex: 確定した仕様・デザインに基づく実装

## 基本フロー

1. 要件定義
2. 画面・機能設計
3. デザイン設計
4. デザインレビュー
5. デザイン決定記録
6. Codex向け資料作成
7. 実装依頼

## ディレクトリ

- `00_index.md`: 主要ファイルへの索引
- `01_workflow/`: 人間とChatGPTで進めるデザイン作業手順
- `02_resources/`: デザイン素材・参考サイト
- `03_design_system/`: 色、文字、余白、角丸などの設計ルール
- `04_components/`: UI部品カタログ
- `05_reviews/`: UI/UXレビュー観点
- `06_decisions/`: デザイン意思決定ログ
- `07_prompts/`: ChatGPTと使う設計・レビュー用プロンプト
- `08_handoff/`: Codexへ渡す資料テンプレート
- `09_case_studies/`: 過去プロジェクトのデザイン判断事例
- `10_operations/`: 運用レビュー、棚卸し、改善提案

## 管理ルール

- 新しい主要ファイルを追加したら `00_index.md` に追記する。
- ファイル名は英小文字のsnake_caseにする。
- 意思決定ログは `06_decisions/adr/DDD-0000-title.md` 形式にする。
- 実装コード、ビルド設定、依存関係は置かない。
- 確定した判断は会話ログではなくMarkdownに残す。

## 100件以上に増えた時の運用

- 大きなまとめファイルへ追記し続けず、テーマ単位で分割する。
- 探す入口はREADMEではなく `00_index.md` に寄せる。
- 古い判断を消さず、`Status: Superseded` で新しい判断へリンクする。
