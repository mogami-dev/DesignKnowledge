# Case Study: 酒コレ

## App Idea

日本酒のラベルを撮影してコレクションするアプリ。

## Target

日本酒を嗜み、飲んだ銘柄やラベルを記録したい人。

## Core Problem

写真フォルダでは日本酒ラベルの写真が埋もれてしまう。

## Core Value

手軽に・きれいに記録し、あとから見返せる。

## Main Use

撮影 → コレクション → 詳細確認。

## Design Direction

- 写真を主役にする
- Apple Photosに近い一覧性
- カードUIは避ける
- 情報量を抑える
- コレクション感を優先する

## Key Screens

1. ラベル一覧
2. ラベル詳細
3. 撮影
4. 撮影後確認・補正
5. タグ別表示
6. 編集

## Decisions

### 写真一覧スタイル

理由:

- 写真枚数を優先したい
- Apple Photosとの親和性が高い
- コレクション感が強い
- カードUIはAIアプリっぽく見えやすい

### 裏ラベル表示

方針:

- 表/裏を切り替える
- Flip Animationを検討
- 詳細画面の体験として楽しくする

### AI補正

方針:

- 撮影後にAI補正できる
- 表ラベルでAI補正した場合は裏ラベルにも流れを引き継ぐ

## Lessons

- 写真が主役のアプリでは、情報カードより写真一覧が合う場合がある
- データ管理感を出しすぎると、コレクション体験が弱くなる
- デザイン判断は必ず理由を残すべき
